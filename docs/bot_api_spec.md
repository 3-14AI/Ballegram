# Спецификация Bot API

Данный документ описывает открытый Bot API платформы Ballegram, позволяющий сторонним разработчикам интегрировать своих ботов для опросов, модерации, CRM-систем и других задач.

## 1. Регистрация бота

Перед началом работы бота необходимо зарегистрировать в системе для получения уникального токена.

*   **Endpoint:** `POST /bot/register`
*   **Описание:** Создает нового бота и возвращает идентификатор и токен авторизации.
*   **Request Body:**
    ```json
    {
        "name": "MyAwesomeBot"
    }
    ```
*   **Response (200 OK):**
    ```json
    {
        "id": "uuid-bot-id",
        "name": "MyAwesomeBot",
        "token": "bot-auth-token-string"
    }
    ```

## 2. Настройка Webhook

Для получения входящих сообщений и обновлений боту необходимо установить URL для Webhook. Ballegram будет отправлять POST-запросы на этот URL при наступлении событий.

*   **Endpoint:** `POST /bot/{botId}/webhook`
*   **Описание:** Устанавливает или обновляет URL вебхука для бота.
*   **Request Body:**
    ```json
    {
        "webhook_url": "https://yourdomain.com/webhook/endpoint"
    }
    ```
*   **Response (200 OK):** Пустое тело при успехе.

## 3. Формат входящих обновлений (Webhook Payload)

Когда в Ballegram происходит событие, связанное с ботом (например, боту отправлено сообщение или его добавили в группу), сервер платформы отправит POST-запрос на зарегистрированный `webhook_url` с полезной нагрузкой:

*   **Request Header:** `Content-Type: application/json`
*   **Payload Example:**
    ```json
    {
        "id": "event-uuid",
        "action": "new_message",
        "payload": {
            "chat_id": "chat-uuid",
            "sender_id": "user-uuid",
            "text": "Hello bot!"
        }
    }
    ```

## 4. Диспетчеризация событий (Отправка обновлений)

Разработчики могут использовать эндпоинт для тестирования отправки событий на вебхук бота.

*   **Endpoint:** `POST /bot/{botId}/dispatch`
*   **Описание:** Отправляет тестовое обновление на настроенный `webhook_url` бота.
*   **Request Body:**
    ```json
    {
        "id": "test-event-uuid",
        "action": "test_action",
        "payload": {
            "key": "value"
        }
    }
    ```
*   **Response (200 OK):** Пустое тело в случае успешной доставки вебхука.

---
*Примечание: Развитие Bot API продолжится для добавления эндпоинтов отправки сообщений от имени бота и управления правами бота в чатах.*
